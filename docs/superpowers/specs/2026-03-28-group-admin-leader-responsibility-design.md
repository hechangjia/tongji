# 管理员建组与组长维护组信息设计

## 背景

当前 `/admin/groups` 同时承担：

- 创建小组
- 编辑小组名称
- 编辑小组口号
- 编辑小组备注
- 指定 / 更换组长

而 `/leader/group` 目前只是只读展示页，组长无法自己维护本组信息。

这会带来两个问题：

1. 管理员和组长对“小组信息”职责边界不清
2. 组长虽然是本组负责人，却不能自己编排口号和备注

## 目标

把小组相关职责拆开：

- 管理员只负责“组织编制”
  - 创建小组
  - 任命 / 更换组长
- 组长负责“本组展示信息”
  - 维护本组口号
  - 维护本组备注

## 职责边界

### 管理员保留

- 创建小组
- 查看小组列表
- 指定 / 更换 / 移除组长
- 在成员管理中继续维护成员所属小组与角色

### 管理员移除

- 不再在 `/admin/groups` 里维护：
  - 小组口号
  - 小组备注

### 组长新增

- 在 `/leader/group` 里编辑自己所属小组的：
  - `slogan`
  - `remark`

### 组长不能改

- 小组名称
- 组长身份
- 组成员归属
- 其他小组的数据

## 页面调整

### `/admin/groups`

- “新增小组”表单保留：
  - 小组名称
  - 指定组长（可选）
- 移除管理员创建时填写 `slogan` / `remark` 的入口
- 列表区保留：
  - 小组名称
  - 当前组长
  - 成员数
  - 组长任命表单
- 不再提供管理员修改 `slogan` / `remark` 的表单

### `/leader/group`

- 从只读展示页升级为“组长可编辑小组信息”页
- 保留小组概览卡片：
  - 小组名称
  - 组员人数
  - 当前组长
- 新增一个编辑表单：
  - 小组口号
  - 小组备注
- 提交后返回当前页，并展示成功 / 失败提示

## 数据与权限

- 不改 Prisma schema
- 继续复用 `Group.slogan`、`Group.remark`
- 新增 leader 侧 update action
- 该 action 必须校验：
  - 当前用户是 `LEADER`
  - 当前用户已绑定 `groupId`
  - 只能更新自己 `groupId` 对应的小组

## 验收标准

1. 管理员在 `/admin/groups` 只能建组和任命组长，不能再直接编辑口号和备注
2. 组长在 `/leader/group` 可以保存本组口号和备注
3. 未绑定小组的组长仍然看到空态，不会出现编辑表单
4. 非组长和未登录用户不能调用 leader 侧小组更新动作
5. 现有“升为组长必须绑定小组”的成员管理规则继续保留
